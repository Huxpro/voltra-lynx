# Vue Lynx 集成

Voltra **与宿主框架无关**。实时活动(Live Activity)和小组件最终都被生成为
一份 JSON 负载,再由原生代码转换成 SwiftUI / Compose Glance,因此驱动界面的
JS 框架是可以替换的。当你用 [**Vue Lynx**](https://vue.lynxjs.org/) 而不是
ReactLynx 来构建应用时,[快速开始](/getting-started/introduction) 和各平台指南
里的内容依然适用 —— 只有宿主 UI 层会发生变化。

如果你是从 ReactLynx 文档过来的:Voltra 的 API、组件形态、负载格式、原生宿主
以及样式规则都是**完全一样**的。本页只介绍有差异的部分。

## 完全一致的部分(与 ReactLynx 共享)

迁移到 Vue 宿主时,以下内容**无需**任何改动:

| 层 | 内容 | 包 |
|---|---|---|
| 纯 JS | 渲染器、组件、负载、类型 | `@use-voltra/core`、`@use-voltra/ios`、`@use-voltra/android` |
| 桥接 + 客户端 | `useLiveActivity`、`startLiveActivity`、小组件 API、原生模块适配器 | `@use-voltra/lynx` |
| 负载 | 发送给原生的序列化 JSON | **逐字节完全一致** |
| 原生宿主 | iOS Widget Extension / Android Glance 模块 | 不变 —— 见 [iOS](/ios/setup) / [Android](/android/setup) |
| 组件 API | 所有 `Voltra.*` 组件及其 props、样式、事件 | 不变 —— 见 [Voltra 文档 ↗](https://www.use-voltra.dev/) |
| 样式 | Lynx CSS 约定 | 不变 —— 见 [iOS 样式说明](/ios/styling) |

一句话:**第 0–3 层原样复用**,只有 UI 宿主(“应用”层)换了框架。

## 发生变化的部分(仅宿主 UI 层)

| 关注点 | ReactLynx | Vue Lynx |
|---|---|---|
| 构建插件 | `pluginReactLynx()` | 来自 `vue-lynx/plugin` 的 `pluginVueLynx()` |
| 运行时包 | `@lynx-js/react` | `vue-lynx`(外加 `vue` 3.5、`@rsbuild/plugin-vue`) |
| 入口 | `root.render(<App />)` | `createApp(App).mount()` |
| UI 编写 | `.tsx`(JSX) | `.vue` 单文件组件(`<template>`、`@tap`、`:style`) |
| `react` 解析 | 别名 → `@lynx-js/react` | **不**做别名 → 真正的 `react` |

## 1. 安装依赖

<PackageManagerTabs command="install @use-voltra/lynx @use-voltra/core @use-voltra/ios @use-voltra/android vue-lynx vue react" />

`vue-lynx` 提供 Vue 3 运行时和 `createApp`,其 peer 依赖是 `@rsbuild/core` 和
`@rsbuild/plugin-vue`。你还需要安装 `react` —— 它不作为 UI 框架使用,而是因为
Voltra 的组件本质是 `createElement` 工厂,用来构建负载(见
[§4](#4-构建负载与宿主无关))。

## 2. 配置 Rspeedy

把 `pluginReactLynx` 换成 `pluginVueLynx`。固定开发服务器端口,让原生宿主每次都
能找到你的 bundle —— 这一点和 React 配置完全相同:

```ts title="lynx.config.ts"
import { defineConfig } from '@lynx-js/rspeedy'
import { pluginVueLynx } from 'vue-lynx/plugin'

export default defineConfig({
  environments: { lynx: {}, web: {} },
  plugins: [
    pluginVueLynx({ optionsApi: false }),
  ],
  server: { port: 3000, strictPort: true },
})
```

## 3. 配置原生宿主

无变化。实时活动和小组件在原生 OS 扩展中渲染,因此你仍然需要一个嵌入 Lynx 并
注册了 Voltra 原生模块的 iOS 或 Android 宿主。沿用相同的平台指南:

- [**iOS 配置**](/ios/setup):Xcode 工程、CocoaPods、Widget Extension
- [**Android 配置**](/android/setup):Gradle 模块、Glance 小组件

由于两种框架都会编译成在 `:3000` 上提供的 `main.lynx.bundle`,宿主无需任何原生
改动即可加载 Vue 的 bundle —— 只要运行 Vue 应用的开发服务器,而不是 React 的。

## 4. 构建负载(与宿主无关)

这是 Vue 支持的关键所在。Voltra 的 `Voltra.*` 组件就是普通的 `createElement`
工厂,而 `renderLiveActivityToString()`(由 `startLiveActivity` 自动调用)会把
生成的元素树序列化为 JSON。这一步是纯 JS —— 它从不接触 DOM 或宿主渲染器 ——
所以无论外层应用是 React 还是 Vue,产出的负载都是**一样的**。

为了让负载不依赖任何宿主的 JSX 转换,请在普通 `.ts` 文件里**直接**用
`createElement`(不用 JSX)来编写:

```ts title="voltra-payloads.ts"
import { createElement as h } from 'react'
import { Voltra } from '@use-voltra/ios'

export function makeBasicVariants() {
  return {
    lockScreen: h(
      Voltra.VStack,
      { style: { padding: 16, borderRadius: '14px', backgroundColor: '#111827' } },
      h(Voltra.Text, { style: { color: 'white', fontSize: 18 } }, 'Hello from Vue Lynx'),
    ),
  }
}
```

## 5. 在 Vue 组件中驱动

`.vue` 组件保持纯 Vue。它调用的是和 React 一样的 Voltra 客户端 ——
`startLiveActivity`、`stopLiveActivity`、`useLiveActivity`、小组件 API ——
并传入第 4 步得到的 variants:

```vue title="App.vue"
<script setup lang="ts">
import { startLiveActivity } from '@use-voltra/lynx/ios-client'
import { makeBasicVariants } from './voltra-payloads'

async function start() {
  await startLiveActivity(makeBasicVariants(), { activityName: 'demo' })
}
</script>

<template>
  <view class="Screen">
    <view class="Btn" @tap="start">
      <text>Start Live Activity</text>
    </view>
  </view>
</template>
```

入口使用 Vue 的 `createApp` —— Vue Lynx 挂载到页面根节点,因此 `mount()` 不接受
任何 DOM 选择器:

```ts title="index.ts"
import { createApp } from 'vue-lynx'
import App from './App.vue'

createApp(App).mount()
```

## 唯一的细节:Vue 下没有 `react` 别名

`pluginReactLynx` 会在构建期把 `react` 别名为 `@lynx-js/react`;而
`pluginVueLynx` **不会**。所以在 Vue 宿主里,`@use-voltra/ios` 导入的 `react` 会
解析到**真正的** `react` 包(这也是 [§1](#1-安装依赖) 要装它的原因)。这没问题
—— 反而更干净 —— 因为生成负载只需要 `createElement` 来构建普通的元素对象。

`.vue` 文件保持纯 Vue,负载模块保持纯 React 元素工厂,二者只在调用 Voltra
客户端时相遇。由于负载模块使用 `createElement` 而非 JSX,所以不存在 JSX
运行时冲突。
